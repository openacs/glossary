Glossary Package 4.0a Release Notes

by Walter McGinnis
  • There are substantial comments in the code about things that might break in this release. Take a look there for answers if something breaks.

Currently, I hope to have the following done by the final release.

  • UI support for multiple illustrations for a term
  • UI support for associating a glossary with another object
  • UI support for setting a glossary owner other than the creating user
  • a proc to create a glossary associated with a specific object, available to other packages
  • UI for setting owner of glossary other than creating user
  • UI for displaying owner of glossary
  • better handling of navigation for workflow and general-comments
  • should throw an error in workflow and general-comments have not been instatiated

Configuration

  • Make sure your instanciate acs-workflow and general-comments in the following site-node map:
      • <parent-site-node/
    • parent-site-node/acs-workflow-package
    • parent-site-node/general-comments-package
    • parent-site-node/glossary-package
